Core Modules

Method Layer

Turns vague ideas into concrete project definitions, role maps, MVP boundaries, and build stages.

Runtime Layer

For each round, it decides the current stage, one highest-priority task, what not to do, and when to move on.

Delivery Layer

Keeps the project pointed at deployment, docs, GitHub, demos, and actual handoff instead of endless feature drift.

What it solves

No more prompt wandering

Instead of re-explaining the whole project every time, the skill keeps a stable operating model for real product work.

No more feature rabbit holes

It explicitly flags non-blocking detail traps and pulls the build back toward current stage goals and delivery value.

No more “almost done” projects

It treats deployment, packaging, GitHub, and documentation as part of completion, not optional afterthoughts.
