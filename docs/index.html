project-from-idea-to-product
从想法到产品的项目总控 skill

一个面向真实项目交付的 Agent Skill。它不是只负责分析需求，而是负责把模糊的软件 / 系统 / 后台 / 工具想法，持续推进成可开发、可纠偏、可上线、可包装、可交付的完整项目。

它的核心价值不是“多生成一点代码”，而是持续控制项目节奏：先定义、再拆解、再按阶段推进、边开发边纠偏、避免深陷细节，最后把项目真正推到交付成果。

项目定位

不是普通 prompt 集，也不是只会讲方法论的 skill，而是一个帮助你和龙虾把项目真正做成的项目总控器。

核心循环

任务 → 实现 → 验收 → 纠偏 → 下一任务

适配项目

工单系统、后台产品、内部工具、工作流系统、多角色协同项目，以及所有需要最终上线与交付的 AI 产品项目。

GitHub 仓库： https://github.com/HachikoJ/project-from-idea-to-product

项目展示页： https://hachikoj.github.io/project-from-idea-to-product/

核心能力
把“需求分析”“开发推进”“过程纠偏”“上线包装”拉到同一条项目交付主线上，而不是分散成几份互不相干的说明。

项目定义

帮助用户明确目标用户、系统角色、核心场景、主链路、MVP 和暂不做什么。

需求拆解

把模糊想法拆成页面清单、模块结构、状态流转、角色关系与开发优先级。

运行指挥层

持续判断当前阶段，只给一个当前最高优先级任务，并明确当前不要做什么。

外科手术式纠偏

默认优先最小修改，不轻易重构，不顺手改 unrelated 模块，明确修复完成标准。

防深陷细节

及时识别非阻塞优化、未来功能和局部沉迷，把注意力拉回当前阶段目标与交付主线。

交付导向开发闭环

让项目不断朝部署、GitHub、README、教程、展示页这些真实成果推进，而不是停在本地 demo。

运行工作流
不是一次分析完就结束，而是作为项目控制层持续参与每一轮推进。
Step 1

判断当前项目所处阶段。

Step 2

只下发当前唯一最高优先级任务。

Step 3

由龙虾按当前范围执行实现。

Step 4

回到 skill 做验收与纠偏。

Step 5

通过后再进入下一阶段与下一任务。

为什么它不是普通项目规划 skill
普通 skill 多半停在“帮你列需求”，这个项目额外补上了控制项目节奏的关键一层。

不是只会列需求

它更强调先做什么、先不做什么、什么时候停手进入下一阶段。

不是只会讲方法论

它已经内置《匿证》案例抽象，以及“内部售后工单系统”迁移试跑案例。

不是只会推生成

它会持续帮助判断当前任务是否真的通过验收，并防止项目被细节拖垮。
