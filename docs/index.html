Project Director Skill / Runtime Control Layer

project-from-idea-to-product

一个面向真实项目交付的 Agent Skill。它不是只负责分析需求，而是负责把模糊的软件 / 系统 / 后台 / 工具想法，持续推进成可开发、可纠偏、可上线、可包装、可交付的完整项目。

这不是一个普通 prompt 集，也不是一个只会讲方法论的 skill。它更接近一个项目总控器：负责定阶段、控范围、防跑偏、做验收，并持续把龙虾拉回交付主线。

核心模块

方法层

负责把一个模糊想法定义成真实项目，包括角色、主链路、MVP、需求结构与阶段边界。

指挥层

负责判断当前阶段，压缩成一个当前最高优先级任务，明确当前不要做什么，并控制是否进入下一阶段。

交付层

负责把项目从“能跑”继续推进到部署、GitHub、README、教程、展示页等完整成果形态。

为什么它不是普通 skill

不是只会分析

它不是分析完就结束，而是继续参与任务下发、验收判断、跑偏纠正与阶段切换控制。

不是只会列需求

它强调的是“先做什么、先不做什么、什么时候收手进入下一阶段”，而不是把所有功能都提前列满。

不是只会讲方法论

它已经内置《匿证》案例抽象与“内部售后工单系统”试跑样本，证明它能迁移到新项目题目上工作。

核心能力清单
  • 项目定义与 MVP 收敛

  • 运行指挥层与阶段控制
  • 外科手术式纠偏
  • 防深陷细节控制机制
  • 交付导向开发闭环
  • 部署与域名 / 子路径意识
  • GitHub / README / 教程 / 展示页交付意识
这是一个以“把项目真正做成”为目标的 skill 包，而不是一组静态的项目提示词。
