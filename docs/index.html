Agent Skill / Project Director / Delivery Control Layer

project-from-idea-to-product

一个把模糊的软件想法、后台需求、内部系统构想，持续推进成真实可开发、可纠偏、可上线、可交付成果的项目总控 skill。

它不是普通 prompt 集，也不是只会讲方法论的 skill。它真正解决的是：项目怎么立起来、怎么收住范围、怎么边做边纠偏、怎么避免沉迷细节、怎么把龙虾拉回交付主线，最后把项目真正做成。

核心结构

方法层

负责把模糊需求压缩成真实项目：角色、主链路、MVP、页面结构、优先级与开发阶段。

指挥层

负责判断当前阶段、输出当前唯一高优先级任务、明确当前不做什么，并控制什么时候进入下一阶段。

交付层

负责把项目继续拉到部署、GitHub、README、教程、展示页这些真正能交付和复用的成果形态。

它真正解决什么问题

不是不会写，而是不会推进

很多项目失败，不是因为不会编码，而是不会立项、不会收敛、不会分阶段推进。

不是没做，而是做散了

很多项目不是没产出，而是陷进局部细节、未来功能、非阻塞优化，最后没有形成真正可交付成果。

不是少文档，而是少闭环

这个 skill 强调的不是写满文档，而是让需求、开发、验收、纠偏、上线、包装形成同一条交付闭环。
