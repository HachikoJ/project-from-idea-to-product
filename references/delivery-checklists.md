# Delivery Checklists

## 1. 项目启动清单

- [ ] 项目名称明确
- [ ] 目标用户明确
- [ ] 管理角色明确
- [ ] 核心问题明确
- [ ] 主链路明确
- [ ] MVP 明确
- [ ] 暂时不做什么明确

---

## 2. 主链路闭环清单

- [ ] 用户核心动作可以完成
- [ ] 管理端核心动作可以完成
- [ ] 状态流转跑通
- [ ] 详情链路可进入
- [ ] 必要通知可到达
- [ ] 用户能看到处理结果

---

## 3. 纠偏检查清单

- [ ] 现象描述清楚
- [ ] 根因判断明确
- [ ] 修改范围控制住
- [ ] unrelated 模块未被误伤
- [ ] 修复结果可验证

---

## 4. 上线前检查清单

- [ ] 页面路径检查完成
- [ ] API 路径检查完成
- [ ] 静态资源检查完成
- [ ] 子路径部署约束检查完成
- [ ] 域名访问检查完成
- [ ] 进程常驻方式确定
- [ ] 关键链路线上验收完成

---

## 5. GitHub 备份清单

- [ ] 仓库已创建
- [ ] README 已整理
- [ ] .gitignore 已检查
- [ ] docs / 教程文件已补充
- [ ] 图片命名与路径一致
- [ ] 可公开内容与私密内容已区分
- [ ] 远端推送成功

---

## 6. 文档与教程交付清单

- [ ] 用户版教程已整理
- [ ] 截图位置已标注
- [ ] 电脑版 / 移动版已区分
- [ ] 页面说明与截图一致
- [ ] GitHub 下载路径可访问
- [ ] 如有需要，讲师版讲稿已准备

---

## 7. 项目真正完成的最低标准

- [ ] 主链路可用
- [ ] 关键体验可接受
- [ ] 可上线
- [ ] 可访问
- [ ] 可展示
- [ ] 可交付
- [ ] 可复盘
