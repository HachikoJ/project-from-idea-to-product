# project-from-idea-to-product

A delivery-oriented Agent Skill for turning vague product ideas into real, shippable software projects.

It does not stop at requirement analysis. It helps drive the full loop:

- define the project
- break down the MVP
- control phase scope
- guide implementation priorities
- prevent detail rabbit holes
- run acceptance and correction loops
- push toward deployment and delivery

## What makes it different

Most project-planning skills stop at structure.

This one adds a runtime orchestration layer:

- stage detection
- one current highest-priority task
- explicit do-not-do list
- stage exit criteria
- delivery-driven dev loop
- anti-overfitting / focus control

That makes it closer to a project director than a static prompt pack.

## Recommended usage

1. Ask the skill to define the project first
2. Let it narrow the MVP and current stage
3. Let it output exactly one current highest-priority task
4. Use Long Da to implement only that scope
5. Return to the skill for acceptance and correction
6. Move to the next stage only after passing exit criteria

See: `references/how-to-use-this-skill.md`
